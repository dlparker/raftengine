
New cluster
1. Create initial config and save config
   1. create_cluster.py -a OR
   2. create_cluster.py -n 192.168.100.200 192.168.100.214 192.168.100.222
2. On each host that will run a node, prep_local.py pointed at config.
   1. with --reset --hard to remove the working directory files before re-creating them
   2. with --check to compare the log and the config to ensure nothing is out of sync
3. On each host that will run a node, run_server.py pointed at working dir.

Existing cluster
1. Run cluster_status.py with either a uri or a cluster_config file.
2. If servers not running, on target host run start_local.py pointed at config
